# 提示词库 — 分子与光 Skill 开发全过程

## 核心提示词分类

### 一、项目初始化提示词

```
基于 AetherViz Master 框架构建大学物理实验虚拟仿真网页：分子与光

核心原理：每个分子都具有特定的固有频率(f)。当入射光频率等于分子固有频率时，
光子被共振吸收，分子剧烈振动，随后恢复稳定并发射同频率光子。

技术要求：
1. Three.js r134 进行 3D 分子渲染
2. 单文件 HTML 架构，零构建依赖
3. TailwindCSS + KaTeX 辅助
4. 8 种分子模型（CO/N₂/O₂/CO₂/CH₄/H₂O/NO₂/O₃）
```

### 四、光子物理提示词

```
实现光子物理仿真：

光子状态机：
1. traveling：从光源出发，沿 x 轴正方向运动
2. absorbing：碰撞分子后缩小消失（0.35s 过渡）
3. emitted：分子发射的光子，随机方向扩散

碰撞检测：光子距分子中心 < 0.6 时触发
- 频率差 |f_incident - f_natural| ≤ f₀×2% → 共振吸收
- 否则 → 光子穿透

分子振动：阻尼谐振 A(t) = A₀·e^(-1.2t)·sin(10πt)，持续 2.8s
振动后随机方向发射新光子
```

## 迭代提示词（优化过程）

### 去除干扰元素

```
画面中有灰色像素块缓缓移动（背景星空粒子），需要完全移除，
保持画面干净，不要任何背景装饰粒子。
```

### 按钮固定宽度

```
光频选择按钮和选择分子按钮宽度不一致，改为固定宽度 78px，
不要用 min-width，用固定 width，确保不受文字长度影响。
```

### 控制面板对齐

```
两行控件需要完全对齐，使用扁平 flex 布局：
[标签 56px][滑块 320px][数值 78px][标签 52px][按钮组]
不要用嵌套 div，直接在一行 flex 容器内排列。
```

## 提示词使用技巧总结

1. **参数具体化**：不要说"调整相机"，要说"位置(5,3,6)，FOV 55°，阻尼 0.08"
2. **结构描述化**：用 ASCII 图描述布局结构 `[A 56px][B 320px][C 78px]`
3. **约束明确化**：强调"固定宽度"vs"最小宽度"的区别
4. **分步迭代**：先实现功能，再优化 UI，最后清理细节
5. **物理公式化**：用数学公式描述物理行为 `A(t) = A₀·e^(-decay·t)·sin(f·t)`
